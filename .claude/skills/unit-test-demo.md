# 单元测试技能演示

## 概述

这个技能可以自动运行项目的单元测试，并生成详细的测试报告。

## 使用方法

### 方法1：直接调用测试命令

在 Claude 中输入以下命令来运行测试：

```
/test
```

或者使用完整的技能调用：

```
/skill unit-test
```

### 方法2：手动运行测试

如果你想手动运行测试，可以使用以下命令：

```bash
# 运行所有测试
npm test

# 运行测试并生成覆盖率报告
npm run test:coverage

# 生成详细测试报告
npx tsx scripts/generate-test-report.ts
```

## 测试报告内容

技能会生成以下内容：

1. **测试结果摘要**
   - 总测试数
   - 通过数
   - 失败数
   - 跳过数
   - 通过率

2. **失败详情**（如果有）
   - 失败的测试名称
   - 错误信息
   - 预期值与实际值对比

3. **覆盖率报告**
   - 语句覆盖率
   - 分支覆盖率
   - 函数覆盖率
   - 行覆盖率

4. **测试详情**
   - 每个测试的状态
   - 执行时间

## 输出文件

测试报告会保存到以下位置：

- `test-results/test-report.json` - JSON 格式报告
- `test-results/test-report.txt` - 文本格式报告
- `coverage/index.html` - HTML 覆盖率报告

## 故障排除

### 问题：测试运行失败
**解决方案：**
1. 检查是否安装了所有依赖：`npm install`
2. 检查测试文件语法是否正确
3. 查看错误信息并修复

### 问题：找不到测试文件
**解决方案：**
1. 确保测试文件在 `src/` 目录下
2. 确保测试文件以 `.test.ts` 或 `.spec.ts` 结尾
3. 检查 `vitest.config.ts` 中的 `include` 配置

### 问题：覆盖率报告不生成
**解决方案：**
1. 确保安装了 `@vitest/coverage-v8`
2. 运行 `npm run test:coverage` 生成覆盖率
3. 检查 `coverage/` 目录的权限

## 配置自定义

### 修改测试环境

编辑 `vitest.config.ts`：

```typescript
export default defineConfig({
  test: {
    environment: 'happy-dom', // 或 'jsdom'
    // 其他配置...
  }
})
```

### 设置覆盖率阈值

```typescript
test: {
  coverage: {
    thresholds: {
      statements: 80,
      branches: 75,
      functions: 80,
      lines: 80
    }
  }
}
```

### 排除文件

```typescript
test: {
  coverage: {
    exclude: [
      'node_modules/',
      'dist/',
      '**/*.config.*',
      '**/*.d.ts'
    ]
  }
}
```

## 最佳实践

1. **测试命名**：使用描述性的测试名称
2. **测试隔离**：每个测试应该独立运行
3. **Mock 使用**：对外部依赖进行 mock
4. **覆盖率目标**：设置合理的覆盖率阈值
5. **持续集成**：在 CI/CD 中运行测试

## 相关文件

- `vitest.config.ts` - 测试配置
- `src/test-setup.ts` - 测试环境设置
- `TESTING.md` - 详细测试文档
- `scripts/generate-test-report.ts` - 报告生成脚本
