# Test Automation Agent - 使用示例

## 🚀 快速开始

### 基础调用
```
/test-automation
```
或者
```
"运行测试自动化 agent"
```

### 自然语言调用
```
"帮我全面测试这个项目"
"分析测试覆盖率并提供改进建议"
"检查哪些代码没有被测试覆盖"
```

## 📋 使用场景

### 场景 1：日常开发
**触发**: 每次提交代码前
```
"运行测试，确保没有破坏现有功能"
```
Agent 会：
1. ✅ 运行所有单元测试
2. ✅ 检查测试是否通过
3. ✅ 生成简短报告
4. ⏱️ 耗时：约 30 秒

### 场景 2：代码审查
**触发**: PR 提交后
```
"分析这个 PR 的测试覆盖情况"
```
Agent 会：
1. ✅ 运行测试并生成覆盖率
2. ✅ 识别新增代码的测试覆盖
3. ✅ 生成详细报告
4. 💡 提供改进建议
5. ⏱️ 耗时：约 2 分钟

### 场景 3：质量审计
**触发**: 版本发布前
```
"全面审计测试质量，提供改进建议"
```
Agent 会：
1. ✅ 运行完整测试套件
2. ✅ 深度分析覆盖率
3. ✅ 识别测试质量问题
4. ✅ 生成综合报告
5. 📋 提供优先级建议
6. ⏱️ 耗时：约 5 分钟

### 场景 4：调试失败测试
**触发**: 测试失败时
```
"分析这个测试为什么失败"
```
Agent 会：
1. ✅ 运行失败的测试
2. ✅ 捕获详细错误信息
3. ✅ 分析失败原因
4. 🔍 对比预期和实际值
5. 💡 提供修复建议
6. ⏱️ 耗时：约 1 分钟

### 场景 5：新功能测试
**触发**: 添加新功能后
```
"为新增的函数建议测试用例"
```
Agent 会：
1. ✅ 分析函数签名
2. ✅ 识别边界条件
3. ✅ 生成测试模板
4. 📝 提供示例代码
5. ⏱️ 耗时：约 1 分钟

### 示例 3：失败分析
```
$ "分析测试失败原因"

╔══════════════════════════════════════════════════════════════╗
║                      失败分析报告                           ║
╚══════════════════════════════════════════════════════════════╝

❌ 失败测试: formatAmount should handle NaN

📍 位置
├── 文件: src/__tests__/format.test.ts
├── 行号: 25
└── 测试: formatAmount should handle NaN

🔍 错误详情
├── 预期: NaN
├── 实际: 0
└── 错误: expect(received).toBe(expected)

📝 失败原因
函数没有正确处理 NaN 输入，当输入为 NaN 时返回了 0 而不是 NaN

💡 修复建议
在 formatAmount 函数中添加 NaN 检查：
```typescript
if (isNaN(amount)) {
  return NaN;
}
```

🧪 建议的测试
```typescript
it('应该处理 NaN', () => {
  expect(formatAmount(NaN)).toBe(NaN);
});
```
```

## 🆘 故障排除

### Agent 不响应
```
问题: 调用 agent 后没有反应
解决: 检查 .claude/agents/test-automation.md 是否存在
```

### 测试运行失败
```
问题: Agent 报告测试运行失败
解决: 先手动运行 npm test 看具体错误
```

### 覆盖率不准确
```
问题: 覆盖率数字异常
解决: 删除 coverage/ 目录，重新运行
```
