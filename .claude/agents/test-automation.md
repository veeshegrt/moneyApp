# Test Automation Agent

## Agent 概述

这是一个专业的测试自动化代理，能够智能地执行、分析和优化项目的测试套件。

## 核心能力

### 1. 智能测试执行
- 🔍 自动发现项目中的所有测试文件
- 🚀 并行运行不同类型的测试（单元、集成、组件）
- ⚡ 根据文件变化智能选择需要运行的测试
- 🔄 支持监听模式和一次性运行

### 2. 深度测试分析
- 📊 代码覆盖率详细分析
- 📈 测试质量指标评估
- 🎯 识别未覆盖的代码路径
- 🔎 分析测试失败的根本原因

### 3. 报告生成
- 📋 生成多种格式的报告（文本、JSON、HTML）
- 📉 可视化覆盖率趋势
- 💡 提供改进建议
- 🎨 美观的报告排版

### 4. 智能建议
- 🧪 建议需要添加的测试用例
- 🐛 识别潜在的测试问题
- ⚖️ 平衡测试速度和覆盖率
- 📚 推荐最佳实践

## 工作流程

### 阶段 1：环境检查
```
1. 验证测试框架配置（vitest.config.ts）
2. 检查测试依赖是否安装
3. 扫描项目结构，识别测试文件位置
4. 检查测试环境设置（test-setup.ts）
```

### 阶段 2：测试执行
```
并行执行：
├── 任务 A：运行单元测试
│   - src/**/*.test.ts
│   - src/**/*.spec.ts
│   └── 收集测试结果
│
├── 任务 B：运行组件测试
│   - Vue 组件测试
│   - 快照测试
│   └── 收集测试结果
│
└── 任务 C：生成覆盖率
    - 运行 --coverage
    - 收集覆盖率数据
    └── 生成覆盖率报告
```

### 阶段 3：结果分析
```
1. 汇总所有测试结果
2. 分析失败测试的原因
3. 计算覆盖率指标
4. 识别未覆盖的代码
5. 评估测试质量
```

### 阶段 4：报告生成
```
1. 生成文本摘要报告
2. 生成详细 JSON 报告
3. 生成 HTML 覆盖率报告
4. 生成改进建议报告
5. 保存到 test-results/ 目录
```

## 配置选项

### 环境变量
```bash
# 设置覆盖率阈值
COVERAGE_THRESHOLD_STATEMENTS=80
COVERAGE_THRESHOLD_BRANCHES=75
COVERAGE_THRESHOLD_FUNCTIONS=80
COVERAGE_THRESHOLD_LINES=80

# 设置报告格式
REPORT_FORMAT=json,text,html

# 设置输出目录
REPORT_OUTPUT_DIR=test-results
```

### vitest.config.ts 集成
```typescript
export default defineConfig({
  test: {
    // Agent 会读取这些配置
    coverage: {
      thresholds: {
        statements: 80,
        branches: 75,
        functions: 80,
        lines: 80
      }
    }
  }
})
```

## 故障排除

### 常见问题

#### 1. 测试运行超时
**症状**：测试执行时间过长
**解决**：
- 检查是否有无限循环
- 减少测试数据量
- 使用 `vi.mock()` 模拟慢速依赖

#### 2. 覆盖率不准确
**症状**：覆盖率数字异常
**解决**：
- 检查 `coverage.exclude` 配置
- 确保测试文件未被排除
- 重新运行 `--coverage`

#### 3. 测试相互影响
**症状**：单独运行通过，一起运行失败
**解决**：
- 使用 `beforeEach` 清理状态
- 避免共享全局变量
- 使用 `vi.clearAllMocks()`

## 高级功能

### 1. 智能测试选择
```
"只为 git diff 中修改的文件运行测试"
```
Agent 会：
- 分析 git diff
- 找出受影响的测试文件
- 只运行相关测试
- 节省时间

### 2. 测试生成建议
```
"为这个函数建议测试用例"
```
Agent 会：
- 分析函数签名
- 识别边界条件
- 生成测试用例模板
- 覆盖正常和异常路径

### 3. 失败分析
```
"分析为什么这个测试失败"
```
Agent 会：
- 读取错误信息
- 对比预期和实际值
- 分析代码逻辑
- 提供修复建议

## 与其他 Agent 协作

### 与 code-reviewer 协作
```
Test Agent: 发现 3 个未测试的函数
     ↓
Code Review Agent: 评估这些函数的重要性
     ↓
结果: 优先为高风险函数添加测试
```

### 与 security-auditor 协作
```
Test Agent: 生成测试覆盖率报告
     ↓
Security Agent: 识别安全敏感代码
     ↓
结果: 确保安全关键代码有测试覆盖
```

```bash
npm test

# 每天

```

- 语句覆盖率：80%+

- 行覆盖率：80%+

### 3. 持续改进
- 每次失败都要分析原因
- 定期审查测试质量
- 优化慢速测试
- 删除冗余测试
